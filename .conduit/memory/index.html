North star

A phone-first AI coding harness. Drive Claude Code, Codex, and future agents from iOS and Android with per-project tabs for terminal, agent chat, and live preview. The project itself is built under the same harnessed multi-agent dev workflow it ships.

Frozen contracts

Do not amend in a feature PR. Changes go via their own deliberate PR that all in-flight agents must rebase onto.

  1. docs/WEBSOCKET-PROTOCOL.md — wire format between harness (Go) and core (Rust)
  2. docs/AGENT-ADAPTERS.md — how agents are spawned, swapped, and hooked
  3. docs/MEMORY-FORMAT.md — this HTML schema
  4. docs/SESSION-LIFECYCLE.md — checkpoints, watchdog, recovery, agent-swap atomicity

Project-wide decisions

  1. v1 agents: Claude Code + Codex only. Gemini/Aider/Goose/OpenCode are post-v1 and added via TOML+Dockerfile alone.
  2. Harness runs both on local Mac LAN (mDNS) and remote VPS (bearer + TLS via Caddy). Same binary; flag-gated.
  3. iOS distribution: ad-hoc signed IPA → GitHub Release. No TestFlight in v1. Sideload via AltStore / Sideloadly.
  4. Android distribution: signed APK → GitHub Release. No Play Console in v1.
  5. Memory format is HTML, not Markdown. Reason: renders directly in the mobile browser view, supports structured sections.
  6. Two adapter directories on purpose: .conduit/agents/ for dev-time (swe-swe consumed), agents/ for production (conduit-harness consumed). Avoids dev-tooling churn breaking the shipped product.
  7. No web client. The swe-swe browser UI already works against our server because we keep the WebSocket framing byte-identical.
  8. Watchdog does NOT auto-restart crashed agents by default. User taps Resume. Avoids agents looping forever burning credits.

Conventions
